    • 一：ACL概述

    访问控制列表（ACL）

    • 读取第三层，第四层包头信息
    • 根据预定好的规则对包进行过滤

    访问空控制列表的处理过程

    • 如果匹配第一条规则，则不再往下检查，路由器将决定该数据包允许通过或拒绝通过
    • 如果不匹配第一条规则，则依次往下检查，知道有有任何一条规则匹配
    • 如果最后没有任何一条规则匹配，则路由器根据默认的规则将丢弃该数据包

    访问控制列表的类型

    • 标准访问控制列表基于源IP地址的过滤数据包
    • 扩展访问控制列表基于源IP地址，目的IP地址，指定协议，端口等来过滤数据包列表号是100~199
    • 命名访问控制列表命名访问控制列表在标准和扩展访问控制列表中使用名称代替表号

    二：标准访问控制列表

    步骤一：标准访问控制列表的创建

    access-list 1 deny 192.168.1.1 0.0.0.0
access-list 1 permit 192.168.1.0 0.0.0.255

    通配符掩码：也叫做反掩码。用二进制数0和1表示，如果某位为1，表明这意味不需要进行匹配操作，如果0表明需要严格匹配。例如：192.168.1.0/24 子网掩码是255.255.255.0，其反码表示可以通过255.255.255.255减去255.255.255.0得到0.0.0.255

    • 隐含的拒绝语句

      access-list 1 deny 0.0.0.0 255.255.255.255

    步骤二：将ACL应用于接口

    接口模式:ip access-group 列表号 in或out

    access-list 1 deny 192.168.1.1 0.0.0.0 或者写为
access-list 1 deny host 192.1681.1  
    access-list deny 0.0.0.0 255.255.255.255或者写为
access-list deny any

    步骤三：删除已建立的访问控制列表

    no access-list 列表号   //全局配置模式

    步骤四：接口上取消ACL

    no ip access-group 列表号 in或out

    步骤五：查看访问控制列表

    show access-lists //特权模式

    三：扩展访问控制列表

    可以根据源IP地址，目的IP地址，指定协议，端口等过滤数据包

    • 扩展访问控制列表号:100-199
    • eq等于，lt小于，gt大于，neq不等于
    • 扩展ACL应该应用在离源地址最近的路由器上

    扩展访问控制列表案例

    • 例子1
    access-list 101 permit ip 192.168.1.0 0.0.0.255 192.168.2.0 0.0.0255
(允许192.168.1.0网络访问192.168.2.0网络的所有的服务）   //全局模式
access-list 101 deny ip any any //拒绝所有的服务  //全局模式
    • 例子2
    access 101 deny tcp 192.168.1.0 0.0.0.255 host 192.168.2.2 eq 21
(拒绝192.168.1.0网段访问192.168.2.2的TCP的21端口   //全局配置模式
access-list 101 permit ip any any (允许访问所有）
    • 例子3
    access-list 101 deny icmp 192.168.1.0 0.0.0.255 host 192.168.2.2 echo 
拒绝192.168.1.0 ping 192.168.2.2

    • 删除扩展ACL

      no access-list 列表号

      注意：扩展与标准ACL不能删除单条ACL语句，只能删除整个ACL

    四：命名访问控制类表

    • 命名访问控制类表可以配置标准命名可以配置扩展命名

    • 命名访问控制列表的ACL语句默认第一条为10，第二条为20，以此类推

    • 命名ACL可以删除单条ACL语句，而不必删除整个ACL。并且命名ACL语句可以有选择的插入到列表中的某个位置，使得ACL配置更加方便灵活

    • 标准命名ACL的配置

      全局：ip access-list standard 名字
permit hsot 192.168.1.1
deny deny
      ip access-group 名字 in或者out  //命名ACL应用于接口

    • 扩展命名ACL的配置

      全局：ip access-list extended 名字
deny tcp 192.168.1.0 0.0.0.255 host 192.168.2.2 eq 80 //拒绝1.0网段访问2.2的WEB服务
permit ip any any
